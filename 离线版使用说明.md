# 离线版使用说明

## 打开方式

直接打开：

`outputs/travel-journal-mvp/index.html`

这个页面是单文件线下网页，不需要启动服务器。

## 推荐使用流程

1. 先新建一次真实的过往旅行。
2. 从“基本信息”开始补充目的地、日期、同行人和出行动机。
3. 到“轨迹节点”里记录从哪里到哪里、交通方式、核心地点、地图坐标和当天发生的事。
4. 新增轨迹节点会出现在最上方，并自动沿用已有节点日期。
5. 在每个轨迹节点里用一个字段补充衣食住行、花费、天气穿着、体力和路线提醒。
6. 坐标格式使用“纬度,经度”，可以从地图 App 复制，例如：35.0116, 135.7681。
7. 到“重要事项”里记录跨越整趟旅行的意外、踩坑和下次提醒。
8. 到“回忆线索”里写高光、转折、人物、感官记忆和照片素材备注。
9. 到“感悟总结”里写这趟旅行对自己的影响。
10. 点击“生成草稿”。
11. 导出 Markdown 或打印为 PDF。

## 备份和恢复

线下版会自动保存到当前浏览器，但仍建议定期备份。

- “导出记录”：导出完整 JSON，可用于恢复。
- “导入备份”：导入之前导出的 JSON。
- “导出游记”：导出当前旅行的 Markdown 文本。
- “打印/PDF”：调用系统打印，可选择另存为 PDF。

## 当前边界

当前版本先专注“过往旅行复盘”。

暂时不做：

- 出发前计划。
- 旅行中实时记录。
- 云端同步。
- 自动照片墙。
- 真实地图。

照片和地图后续会作为素材层接回到某个轨迹节点里。
