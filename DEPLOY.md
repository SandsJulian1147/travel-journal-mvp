# 正式部署说明

这个项目是纯静态网页，核心入口是 `index.html`。当前版本不需要后端、不需要数据库、不需要构建命令。

## 推荐发布方式

1. 在 GitHub 创建一个空仓库，例如 `travel-journal-mvp`。
2. 把本目录内容推送到该仓库。
3. 在 Vercel 里选择 `Add New Project`。
4. 导入这个 GitHub 仓库。
5. Framework Preset 选择 `Other`。
6. Build Command 留空。
7. Output Directory 留空或填 `.`。
8. 点击 Deploy。

## 数据说明

当前版本使用浏览器本地存储：

- 每个用户的数据只保存在自己的浏览器里。
- 文件版、本地预览版和线上版的数据互不自动同步。
- 换电脑、换浏览器或换域名时，需要先导出 JSON 备份，再导入恢复。

## 后续升级方向

如果需要多人共享、跨设备同步、登录账号、云端照片存储，需要在后续版本加入后端数据库和用户系统。
